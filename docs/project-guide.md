# 프로젝트 지침서

## 기본 원칙

- 답변 마지막에는 다음 계획에 대한 요약 설명을 제공한다.
- 프로젝트 진행 중 이 지침서는 수정될 수 있으며, 변경사항을 무시하지 않고 작업 기준으로 반영한다.
- 사용자의 답변 품질을 높이기 위해 작업 전후로 필요한 질문을 간단히 던지고, 답변을 통해 방향성을 조정한다.

## 작업 방식

- 사용자가 제공하는 프롬프트와 자료를 기준으로 작업한다.
- 불확실한 요구사항은 임의로 확정하지 않고, 가능한 경우 짧은 질문으로 확인한다.
- 산출물은 프로젝트 목적에 맞게 명확하고 재사용 가능하게 정리한다.

## 현재 상태

- 초강스 내부 홍보용 웹페이지 기본 버전 제작 완료.
- Figma 레퍼런스 디자인을 참고한 별도 버전 제작 완료.
- 참석자 제공용 안내 문서 제작 완료.
- GitHub Pages 공개 배포 완료.
- 추후 일정, 장소, 참가비, 신청 방법, 정원은 placeholder 상태로 유지한다.
